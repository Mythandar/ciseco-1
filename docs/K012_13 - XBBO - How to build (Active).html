K012/13 - XBBO - How to build (Active)

Details
Category: Ciseco product documentation
Last Updated on Thursday, 10 April 2014 08:08

 

NOTE: The kit is now supplied with a 22K resistor for the pullup, this is mounted in the middle next to where is marked "pullup".

The XBBO is a breakout board for XBee shaped radios such as the XRF and RV-XV.  It is supplied as a kit of parts in two variations; passive and active.

The Passive XBBO includes XBEE headers, serial header and a capacitor, and must be supplied with a regulated 3.3v

For the Passive XBBO you will need to install two jump links. A link is need to bypass the diode, this is labelled "PASSIVE BYPASS RX" on the silk screen. A second link is needed on the jumper header in position 2, two holes above silk screen.

 

The Active XBBO kit also has logic conversion components and voltage regulator to allow it to be powered with 5v.

To build the active kit, start with the components closest to the board; resistors & diodes.  Note the polarisation line on the diode must correspond to the line marked on the board.  All resistors are  1k (Not 4.7k as shown in the photos!) and can be fitted any way around.  The LEDs need to be fitted with the negative lead (the short one) towards the connector edge of the board and positive lead (the longer one) towards the resistor.  The 0.1uf capacitor can be fitted any way around.  

The voltage regulator needs to be laid down snug against the board with the flat side upwards as per the photo.  The 1uf capacitors are polarised and should be fitted with the negative side (shorter lead and white stripe on casing) next to the XBee header.  Unless you are using a Roving Networks XV (see note below), these need to be  fitted laying down against the board.

The 5 pin serial header  is designed to go across the Tx, Rx, MCPWR, Gnd and RST pins.  You may substitute a longer header if you wish to make other pins available on a single header (very handy for breadboard connections!). 

 

The Xbee headers are easer to solder on to the board if you push them on to your XBee compatible radio first.  You will then know they are strait and will line up properly!  Fit the jumper pins and jumper.  

 

JUMPER SELECTION

The jumper can be in position 1 or 2. In small text on the board you will see
1=MCUPWR or BATTERY
2=PASSIVE FROM HEADER or BATT PWR MODULE
If you have an active version (led's, regulators etc) and want to power from the 5 pin (xino header) then select position 1
If you have an active version and want to power it AND a connected Xino then connect power to the VIN/GND pads on the XBBO and select position 1 (this will power your xino at 3.3v)
If you have a passive version (no regulators just PCB, headers & 0.1uf) then select position 2
 

 

 

When connecting the power leads feed the cables down through the board and backup again for strain releif.

 

The passive kit is similar, but with less components.  Note that a wire jumper needs to be used where the diode is marked on the board.

 

Pull up resistor

The XBBO is supplied with a 1k pullup resistor to the Tx line of the XRF.  This works with most setups, but in some situations it can cause issues.  If your Arduino board already has inline resistors on the Tx line then the XRF may not get a high enough voltage to trigger a logic high.  If you experience this, change the 1k pull up resistor for something larger.  Somewhere between 4k7 and 50k should be fine.

 

 

Note about using Roving Netowrks XV in the Active XBBO

 

The underside of the XV is fitted with the radio gear which will clash with components in the active version of the XBBO.  Whilst this will work, the XV will sit slightly proud of the header connectors.  To avoid this, the capacitors should be mounted underneath the PCB.  

NOTE! Well spotted by Denham, this picture the XV is the wrong way round

The body of the XV is also longer than a standard XBee, and will collide with the jumper pins.  These should either be bent at right-angles away from the PCB, or else a link permanently soldered in place of the jumper
