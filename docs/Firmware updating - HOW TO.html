Firmware updating - HOW TO

Details
Category: Radio basics (XRF, URF, ERF & SRF)
Last Updated on Thursday, 16 June 2016 14:24

Please read carefully

 

This article shows you how to upload different firmware to your XRF.

We strongly suggest you don't upgrade unless you really need to, as an incomplete upgrade means the device may be unable to run its software.

Firmware uplaod's need to be done a 9600 baud, if you have change the baud of you radio to something else you will need to set it back to 9600 before proceding.

 

For details on firmware files please see the following article All XRF Firmware Updates

 

The process works as follows:

  • The uploaded file is loaded into high memory

  • You then verify to make sure it matches the file on your computer (at this point you are still running the existing firmware)

  • When you press commit the XRF copies the contents into "working memory" and restarts after a second or two. Any power interruption or noise in your circuit at the "commit point" could cause problems. If you get any type of error during the program or verify stage DO NOT press commit.

 

The images below show .hex file but all radios shipped since May 2012 use .bin files. The procedure is exactly the same.

 

 

How to upload

 

Select your com port

 

 

 

Go to the "upload firmware" tab

 

 

 

Browse to the location you downloaded the HEX image to, or type in its location

 

 

 

Press 'Program'; the text will grey out and the progress bar will rise (takes around 60 secs)

 

 

 

Press 'Verify'. This checks that what is has been uploaded matches the file contents

 

 

 

Once finished, you should get a message bottom left saying the verify was successful

 

 

 

Press 'Commit', wait a few seconds, go to the general tab and press the download config button. You will see that the version has changed.
