Using an ARF as a LLAP repeater

Details
Category: LLAP - Lightweight Local Automation Protocol
Last Updated on Tuesday, 03 September 2013 12:06

Why use an ARF as a LLAP repeater?

The XRF range is fantastic, however I'm begining to use the SRF a lot especially the RFu (tiny Arduino + radio, link here). The range is lower when used with the defualt compact chip antenna (you can add a wire whip to most of our radios to improve range see here). So what I thought it would be useful to do was add some extra range to my network by adding a repeater station. I could have used a standard XRF but thought why not go the extra mile (pun intended) and use an ARF instead (see here).

The repeater I'm building simply listens to two different PANID's (a group of radios), anything broadcast on one network group is broadcast on the other. It's a great way to extend networks by creating a celluar type arrangement.

 

What do we need?

An ARF and a way to power it...that's really it.

Here's what I built, it's a mini module (see here) without the DC jack (the ARF is longer than the XRF) this is all I need to power the repeater.

 

Configuring the radios and the repeater

We need a way of talking and testing, for this I'm using what's on my desk, an Explorer Plus (see here) and an SRF Stick (see here). I've got them both plugged into my PC.

How I'm setting this up is so I have 2 new networks, there's 1111 and 2222, the 2222 network is the one I want to repeat to the 1111 network, be aware the default PANID for shipped devices is 5AA5 so these will be ignored by anything on the 1111 or 2222 networks.

I set the SRF stick PANID to 1111 by doing the following in XCM, I'm typing directly into the larger window the following (you could use any terminal software)

+++ (I wait for OK)

ATID1111 (enter)

ATWR (enter)

ATAC (enter)

ATDN (enter)

I set the Explorer Plus radio to a PANID of 2222 (you need the red jumper on "SRF")

+++ (I wait for OK)

ATID2222 (enter)

ATWR (enter)

ATAC (enter)

ATDN (enter)

I then plug the ARF into the Explorer Plus and change the jumper to "XRF" this then connects serial to the 2x10 way socket). I need to set four things, the default PANID (1111) and the secondary PANID (2222) and turn on repeater mode (ATNT2). The last part of the jigsaw is the TX to RX delay. It's sometimes useful to have a delay between the repeats to allow other traffic to complete or prevent multiple repeaters repeating at the same time. I've set mine for an incredibly long 50ms, I can tweak it later if I need to.

+++ (I wait for OK)

ATID1111 (enter)

ATI22222 (enter)

ATNT2 (enter)

ATTR50 (enter)

ATWR (enter)

ATAC (enter)

ATDN (enter)

 

That's it!

 

Testing

I put the ARF into the mini module and give it power (I'm using a spare 5v main PSU). I opened up two XCM windows, one connected to the SRF Stick and one to the Explorer Plus (back in SRF jumper mode) and simply typed radom chars into one and see them repeated to the other). I'm happy, time for a beer :)

 

Limitations

It's half duplex. This means the radio can only be recieving or transmitting it cannot do both simulaneously, for that you would need to use two radios. That's really easy to do, simply connect RX to TX and TX to RX on a pair of radios). Because I added the 50ms delay streaming serial would look very disjointed but because I'm only having LLAP devices on repeated side it's perfect for my intended use.
