Using the XRF as a CC1110 device

Details
Category: Radio basics (XRF, URF, ERF & SRF)
Last Updated on Monday, 13 June 2016 15:26

What do we mean?

The XRF comes with serial firmware that allows easy communication with RS232 devices. This is what 99% of our customers want to do. However, there's a small group who are "hardcore" and want to code the 8051 micro from scratch. The XRF gives you the chance to do this from a hardware perspective. How you do it is down to the TI documentation and your own skills.

I'm hardcore, experienced and up for it!

Ok, if you're still interested (and don't want to pay TI $649 to have a go at programming the CC) then you will need:

  1. An XRF - https://www.wirelessthings.net/
  2. A xino basic for XRF (or make something similar) - https://www.wirelessthings.net/
  3. A Texas Instruments CC Debugger - http://uk.farnell.com/texas-instruments/cc-debugger/prog-debugger-for-rf-soc/dp/1752232?Ntt=cc+debugger
  4. Kickstart version of IAR - C compiler - http://www.iar.com

Note the only way to get Ciseco firmware back onto a radio module after erasing it using the TI CC-Debugger would be to retrun the module to Ciseco.
