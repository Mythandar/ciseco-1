XRF modules losing PANID - reverting to default

#10368 1 year, 1 month ago
Hi all

I've been having some issues with XRF modules losing their PANID configuration after losing power.
I used the commands:

+++ (enter command mode)
ATID [panid] (set PANID)
ATAC (apply changes)
ATWR (write the changes to flash)

There was a 150ms delay between each command and I seemed to get the correct number of OKs returned for the number of commands sent. So it looked like the commands executed correctly.

I was configuring them using realterm on windows 8.1 and have also tried on windows 7 much to the same effect. I didn't seem to get these issues when I set up SRF sticks using the same commands.

Does anyone have any idea of what's going wrong with the PANID settings or is it something I'm doing with the commands I'm sending that's wrong?

Re: XRF modules losing PANID - reverting to default

#10369 1 year, 1 month ago
There was a 150ms delay between each command and I seemed to get the correct number of OKs returned for the number of commands sent. So it looked like the commands executed correctly.


It sounds like you are entering the commands via your own program rather than typing into the a terminal window or using
XCM.

If that is the case then you need to verify in your program that you get an OK after each command - "seemed to get the correct number of OKs returned for the number of commands sent" is not good enough. If the ID command is not accepted OK or the WR command is not accepted then the new PANID will not be saved.
